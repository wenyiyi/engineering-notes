# Illustration outline

Visual direction: friendly hand-drawn engineering sketch notes, warm off-white paper, navy ink, coral/orange highlights, teal accents, simple characters and strong visual hierarchy. Images should explain one idea at a glance. Keep labels to a few short English words so the same assets work in Chinese and English editions.

1. Cover — a search box opening into an Elasticsearch “library”.
2. MySQL and Elasticsearch division of labor.
3. A book-title search showing tokenization and relevance.
4. Inverted index: term dictionary to posting lists.
5. Text vs keyword field choice.
6. Query journey: analyze, retrieve, score, highlight.
7. Near-real-time write path: buffer, refresh, segment, merge.
8. Shards, routing, coordinating node, fan-out and merge.
9. Primary and replica failover.
10. MySQL-to-Elasticsearch reliable synchronization.
11. Practical decision map: when Elasticsearch is and is not a fit.
